# Guía rápida para agentes

Antes de modificar el producto, leer `docs/agent/README.md`. Esa carpeta es la
fuente de verdad sobre la visión, el alcance, la arquitectura y el roadmap.

Reglas esenciales:

- Mantener la experiencia mobile-first y en español rioplatense.
- No inventar detalles de la cita: registrar preguntas abiertas y esperar la
  definición del usuario.
- Favorecer componentes simples y datos locales mientras no se necesite un
  backend.
- Conservar compatibilidad con el build estático de GitHub Pages.
- Ejecutar `npm run build` antes de dar una tarea de código por terminada.
- Actualizar la documentación cuando cambie una decisión de producto o técnica.
