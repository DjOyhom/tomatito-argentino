# Base de conocimiento del agente

Esta carpeta es la fuente de verdad para continuar **Tomatito Argentino** sin
perder contexto entre sesiones.

## Orden de lectura

1. [`product-brief.md`](product-brief.md): qué experiencia estamos creando.
2. [`experience-principles.md`](experience-principles.md): cómo debe sentirse.
3. [`architecture.md`](architecture.md): cómo está construida la aplicación.
4. [`roadmap.md`](roadmap.md): qué sabemos, qué falta decidir y próximos pasos.
5. [`decisions.md`](decisions.md): decisiones consolidadas y su motivo.

## Cómo mantener esta documentación

- Actualizar el brief si cambia el objetivo o el público.
- Mover una pregunta abierta del roadmap a decisiones cuando el usuario la
  responda.
- Registrar cambios técnicos importantes en `decisions.md`.
- Documentar solamente comportamiento confirmado; no convertir ideas en
  requisitos sin aprobación.
- Mantener los textos breves, accionables y libres de datos sensibles.
